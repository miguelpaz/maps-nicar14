# Tools #
* [QGIS, for using shapefiles, joining data, and lots of other things](http://www.qgis.org/en/site/)
* [Leaflet, for making slippy maps for the web](http://leafletjs.com/)
* [MapBox, for almost everything](https://www.mapbox.com/)
* [Google Maps API, the granddaddy of them all](https://developers.google.com/maps/)
* [D3, for creating SVG maps for the web](http://d3js.org/)
* [MapStack, for designing quick, custom map images](http://mapstack.stamen.com/)
* [CartoDB, for creating/hosting data-driven web maps](http://cartodb.com/)
* [Kartograph.js, for building SVG maps for the web](http://kartograph.org/)
* [geojson.io, for browsing/editing a geodata file](http://geojson.io/)
* [Google Maps styling wizard, for creating custom background styles for the Google Maps API](http://gmaps-samples-v3.googlecode.com/svn/trunk/styledmaps/wizard/index.html)

# File converters #
* [ADC4GIS converter, for converting to/from GeoJSON](http://ogre.adc4gis.com/)
* [Distillery, for converting GeoJSON to TopoJSON](http://shancarter.github.io/distillery/)
* [Mapshaper, for simplifying or converting shapefiles, GeoJSON, and TopoJSON](http://www.mapshaper.org/)

### Data sources ###
* [US Census Bureau, for detailed data about the US (mostly shapefiles)](http://www.census.gov/geo/maps-data/data/tiger.html)
* [Natural Earth, for downloads of basic natural/political boundaries for the entire world](http://www.naturalearthdata.com/)
* [OpenStreetMap, for detailed, mappable data about almost every place in the world](http://www.openstreetmap.org/)
* [The Los Angeles Times Boundary Service (for Los Angeles)](http://boundaries.latimes.com/sets/)
* [The TribApps Boundary Service (for Chicago)](http://boundaries.tribapps.com/)
* [The MinnPost Boundary Service (for Minnesota)](http://boundaries.minnpost.com/)
* Try your local/state government.  Many of them have GIS sites where you can download detailed files of districts, roads and more.
